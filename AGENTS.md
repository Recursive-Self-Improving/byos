# Lessons

- `github.com/coreos/go-oidc/v3 v3.20.0` requires `golang.org/x/oauth2 v0.36.0`; the lower tracker baseline `v0.30.0` cannot be selected under Go MVS.
- `modernc.org/sqlite v1.53.0` pulls `modernc.org/libc v1.73.4`, which requires `golang.org/x/sync v0.20.0`; the lower tracker baseline `v0.18.0` cannot be selected under Go MVS.
- Go `embed` cannot reference parent directories. Root `migrations/*.sql` are embedded by the root `migrations` package and consumed by `internal/store`.
- `server.trusted_proxies` accepts IP addresses or CIDR prefixes. Web security treats `X-Forwarded-Proto: https` as authoritative only when `RemoteAddr` matches that configured set; otherwise forwarded headers are ignored.
- Administrator Web-password and API-key throttling share a keyed client-source identity. `X-Forwarded-For` is accepted only from configured trusted peers, parsed right-to-left to the first untrusted hop, and malformed trusted chains fail closed before credential evaluation.
- Railway Dockerfile start commands run in exec form, so `$PORT` expansion requires an explicit `/bin/sh -c "exec …"`; the deployment uses one persistent `/data` volume and trusts Railway proxy peers in `100.0.0.0/8` via `deploy/railway.yaml`.
- Railway rejects Dockerfiles containing a Docker `VOLUME` instruction; keep Compose on root `Dockerfile` and point `railway.json` to `Dockerfile.railway`, with persistence supplied only by the Railway Volume mounted at `/data`.
- Railway mounts persistent volumes as `root`; a Dockerfile `USER` set to a non-root UID cannot chmod or write `/data`. The dedicated Railway image must run as UID 0 while the application enforces owner-only directory and database modes.
- Browser form POSTs under `Referrer-Policy: no-referrer` can carry `Origin: null`, which Gorilla CSRF rejects even for same-origin forms. The administrator UI uses `Referrer-Policy: same-origin` so CSRF origin verification retains same-origin provenance without leaking referrers cross-origin.
- `oidc.NewVerifier` constructed manually defaults to `RS256` unless `SupportedSigningAlgs` is set. xAI discovery advertises `ES256`, so the discovered `id_token_signing_alg_values_supported` values must be propagated into the verifier.
- Go `1.26.4` is affected by `GO-2026-5856` in `crypto/tls` and `GO-2026-4970` in `os`; require Go `1.26.5` or newer for source builds. Fresh `golang:1.26-bookworm` builds currently resolve to `1.26.5`, but the tag remains mutable.
- Managed Responses reconstruction accepts canonical item types, so OpenAI easy input messages must be normalized to `type: "message"` and canonical content blocks before `sessions.Prepare`; otherwise valid array input is misclassified as an internal failure.
- OpenAI Responses `function_call_output.status` does not allow `"failed"`; Anthropic `tool_result.is_error` must not be translated to that value. Preserve `tool_choice.disable_parallel_tool_use`, but do not forward Anthropic adaptive-thinking effort to `grok-4.5`: xAI documents `reasoning.effort` support only for `grok-4.3`; request the reasoning summary without an effort instead.
- Large Anthropic coding-agent payloads contain many XML-like tags. Go's default JSON HTML escaping expands each `<`/`>` to `\u003c`/`\u003e`; the attached 99,037-byte request became a 103,483-byte upstream body. Final xAI wire encoding must use `json.Encoder.SetEscapeHTML(false)`; Anthropic-only metadata should be dropped and function tools should default `tool_choice` to `auto`.
- The BYOS rebrand is an intentional fresh-install cutover: Go module/command `byos`, `BYOS_*` secrets, `byos_` downstream keys, `byos_admin_*` cookies, `byos.db`, `byos-data`, and BYOS cryptographic domains have no legacy fallbacks or migration. Account/API-key vocabulary and xAI/Grok provider protocol identifiers remain unchanged; historical plan bodies retain old names under an annotation.
- Public OAuth callback exceptions must be registered only while that provider's callback flow is enabled. Bound `RawQuery` before parsing, validate exact state/code shapes, and apply source throttling; BYOS uses one SQLite connection, so unauthenticated random-state lookups otherwise serialize with normal traffic and amplify denial of service.
- Provider stream limits bound bytes received, not cumulative allocation work. Accumulate text, reasoning, and tool-argument deltas with amortized buffers rather than immutable-string `+=`; token-granularity streams otherwise turn bounded output into quadratic copying and GC pressure.
- Devin CLI OAuth accepts HTTP loopback redirects, not arbitrary public HTTPS callbacks: a public `devin.oauth.callback_origin` produces Devin's `Invalid redirect URI`. Remote deployments advertise `http://127.0.0.1:59653/callback` and complete by submitting the browser's full localhost callback URL through the authenticated Web/Admin API flow; request and forwarded headers are never callback-origin inputs.
- xAI `/billing` is the authoritative monthly quota fetch: never persist a `/billing?format=credits` weekly-only response as a successful snapshot because it erases the last monthly limit and reset. A fresh weekly period may omit `creditUsagePercent`, which means `0%`; monthly failures retain the latest complete snapshot as stale.
- xAI model discovery should fall back from `/models-v2` to `/models` on upstream `5xx` responses as well as `404` or schema incompatibility; authentication failures remain terminal and must not trigger fallback.
- Dashboard readiness follows the configured default model, not mere account presence. When enabled `ready` accounts exist but readiness is false, the Web UI must report the default model as unavailable rather than claiming account setup is required; other providers can remain routable.
- Generic provider rate-limit backoff advances once per completed cooldown window, not once per concurrent in-flight `429`. Reuse an active deadline atomically and retain its level after expiry until a successful request resets it; otherwise one burst can jump an xAI account to a persisted 30-minute cooldown, especially visible on Railway volumes.
- The default fixed model catalog has seven public names: `grok` → `grok-4.5`, `glm` → `glm-5-2`, `swe` → `swe-1-7`, plus canonical `grok-4.5`, `glm-5-2`, `swe-1-6`, and `swe-1-7`. Full-catalog tests and active documentation must track `config.Default().Models.Entries`; CI uses `go test -count=1` so restored Go test caches cannot mask rerun behavior.
